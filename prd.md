# Product Requirements Document (PRD)

## 1. 제품 개요

### 제품명

**FanCut AI**

### 한 줄 설명

아이디어 한 줄을 입력하면 **플롯 기획 → 컷 이미지 생성 → 컷 영상 생성 → 최종 mp4 병합**까지 이어지는 숏폼 제작 스튜디오.

---

## 2. 문제 정의

현재 숏폼 제작 과정은 아래 문제를 갖습니다.

1. 초보자는 장면 단위 플롯을 짜기 어렵다.
2. 이미지 생성과 영상 생성이 서로 분리되어 있어 흐름이 끊긴다.
3. 컷 간 캐릭터와 화풍을 맞추기가 어렵다.
4. 생성 결과를 최종 mp4로 묶는 마무리 작업이 번거롭다.

---

## 3. 목표

### 제품 목표

* 한 번의 입력으로 15~30초 숏폼 제작 플로우를 연결한다.
* 플롯, 이미지, 영상 생성 단계를 한 서비스 안에서 처리한다.
* 컷 단위 수정/재생성으로 결과 완성도를 올릴 수 있게 한다.
* 최종 결과물을 mp4로 다운로드하고 업로드까지 이어지게 한다.

### 해커톤 목표

* 필수 3단계인 **AI 플롯 기획 / AI 이미지 생성 / AI 영상 생성**을 실제 데모 플로우로 보여준다.
* 가산 과제로 **영상 병합/편집**을 선택해 최종 결과물 완성도를 높인다.

---

## 4. 현재 MVP 범위

### 포함

1. 아이디어 입력
2. 5~10컷 플롯 자동 생성
3. 컷별 장면 정보 수정
4. 컷별 이미지 생성, 선택, 재생성
5. 컷별 3~5초 영상 생성
6. 최종 mp4 병합 및 다운로드
7. YouTube 업로드, TikTok mock 업로드

### 제외

1. BGM 합성
2. 자막 합성
3. 정교한 타임라인 편집기
4. 창작자 보상 체계
5. 실제 TikTok 업로드

---

## 5. 현재 구현 방식

### 플롯

* Gemini로 자유 입력을 구조화된 5~10컷 JSON으로 생성
* 필요 시 Google Search 기반 IP 리서치 보강
* 프로젝트 단위 story summary, style bible, character bible 생성

### 이미지

* Together AI 이미지 생성 API 사용
* 컷별 후보 생성 후 사용자가 하나를 선택
* 컷 재생성 가능

### 영상

* deAPI image-to-video로 컷별 3초 또는 5초 영상 생성
* 컷별로 개별 생성/재생성 가능

### 최종 결과

* ffmpeg로 mp4 정규화 및 병합
* 결과 썸네일 생성
* 업로드용 제목/설명/해시태그 초안 생성

---

## 6. 일관성 유지 방식

현재 MVP는 아래 조합으로 일관성을 관리합니다.

* 프로젝트 단위 `character bible`
* 프로젝트 단위 `style bible`
* 컷별 image/video prompt
* 인접 컷 문맥 반영
* 선택적 첫 컷 레퍼런스 이미지

주의:

* 업로드한 레퍼런스 이미지는 현재 첫 컷 기준으로 우선 활용된다.
* 이후 컷은 character/style guide와 prompt 기반으로 정합성을 맞춘다.
* 완전한 동일 얼굴 lock 또는 모든 컷에 대한 강한 reference chaining을 보장하는 단계는 아니다.

---

## 7. 사용자 플로우

1. 프로젝트 제목과 아이디어를 입력한다.
2. AI가 5~10컷 플롯을 생성한다.
3. 사용자가 컷 내용을 수정하거나 순서를 조정한다.
4. 컷별 이미지를 생성하고 후보를 선택한다.
5. 컷별 영상을 생성한다.
6. 전체 컷을 mp4로 병합한다.
7. 결과물을 다운로드하거나 업로드한다.

---

## 8. 데모 포인트

* 아이디어 한 줄에서 바로 플롯이 나온다.
* 컷별 이미지와 컷별 영상이 단계적으로 이어진다.
* 최종 mp4가 즉시 재생 가능하다.
* YouTube 업로드까지 한 화면 안에서 연결된다.

---

## 9. 리스크와 대응

### 리스크 1. 컷 간 일관성 부족

**대응:** character bible, style bible, 컷별 prompt, 인접 컷 문맥, 선택적 첫 컷 레퍼런스 이미지를 함께 사용한다.

### 리스크 2. 생성 시간 증가

**대응:** 기본 컷 수를 6컷, 기본 길이를 3초로 두고, 컷별 재생성으로 범위를 좁힌다.

### 리스크 3. 외부 API 의존성

**대응:** 환경 변수와 오류 메시지를 명확히 노출하고, 각 단계를 분리해 문제 구간을 빠르게 확인할 수 있게 한다.
