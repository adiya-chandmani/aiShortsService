
# MVP PRD

## 제품명(가칭)

**FanCut AI**
IP 기반 2차 창작 숏폼 영상을 빠르게 만드는 AI 창작 도구

---

## 1. 제품 개요

### 1-1. 배경

자료에 따르면 2차 창작 콘텐츠는 1차 창작물(IP)을 기반으로 생성되는 콘텐츠이며, K-pop, 애니, 게임, 버튜버 영역에서 활발히 생산/소비됩니다. 생산자는 창작 편의성과 보상, 소비자는 탐색성과 후원 경험, IP 홀더는 간접 홍보와 보유 IP 관리 니즈를 갖고 있습니다. 또한 영상 제작 과정은 **플롯 기획 → 컷별 이미지 제작 → Start/End 프레임 기반 영상화**의 흐름을 가지며, 현재 입문자와 경험자 모두 반복 작업, 복잡한 프롬프트, 여러 툴을 오가는 비효율을 겪고 있습니다. 

### 1-2. 문제 정의

현재 2차 창작 영상 제작에는 아래 문제가 있습니다.

1. **플롯 기획이 어렵다**
   초보자는 장면 단위로 캐릭터, 구도, 배경, 조명, 분위기를 설계하기 어렵다.

2. **이미지-영상 전환이 어렵다**
   단순 프롬프트만으로는 컷 간 일관성과 시작/종료 프레임 개념을 잡기 어렵다.

3. **툴이 분절되어 있다**
   기획 툴, 이미지 생성 툴, 영상 생성 툴을 따로 써야 해서 제작 흐름이 끊긴다.

4. **반복 생성 비용이 크다**
   결과물이 “뽑기”처럼 느껴져 여러 번 재시도해야 하며 시간이 오래 걸린다. 

### 1-3. 제품 비전

누구나 짧은 아이디어만 입력하면,
**일관된 캐릭터/화풍을 유지한 15~30초짜리 2차 창작 숏폼 영상**을 손쉽게 만들 수 있게 한다.

---

## 2. 목표

### 2-1. 제품 목표

* 초보자도 **복잡한 프롬프트 없이** 2차 창작 영상 제작 가능
* **플롯 기획, 이미지 생성, 영상 생성**을 한 서비스 안에서 연결
* **일관성 있는 캐릭터/스타일 유지**
* 최종적으로 **공유 가능한 숏폼 결과물(mp4/gif)** 생성

### 2-2. 해커톤 MVP 목표

* 사용자가 한 번의 입력으로 5~10개 컷 플롯을 생성할 수 있다
* 각 컷에 대해 이미지를 생성할 수 있다
* 각 컷을 3~5초 영상으로 생성할 수 있다
* 컷들을 이어 붙여 15~30초 결과물을 만들 수 있다

## 3. 타겟 사용자

### 3-1. 1차 타겟

**입문자 크리에이터**

* 팬아트/팬무비를 만들고 싶지만 프롬프트 작성이 어렵다
* 스토리보드 경험이 없다
* 여러 생성 툴을 조합하는 것이 번거롭다

### 3-2. 2차 타겟

**경험자 크리에이터**

* 반복 작업을 줄이고 싶다
* 컷 간 캐릭터 일관성과 제작 속도를 높이고 싶다

### 3-3. 간접 이해관계자

* **소비자**: 콘텐츠 탐색, 감상, 반응, 후원
* **IP 홀더**: 간접 홍보, IP 노출 확대, 밈 기반 확산

## 4. 핵심 사용자 시나리오

### 시나리오 A. 초보 팬 크리에이터

1. 사용자가 “슬램덩크 느낌의 농구 경기 하이라이트, 뜨거운 청춘 감성” 같은 짧은 문장을 입력
2. 서비스가 자동으로 6개 컷 플롯 생성
3. 사용자가 캐릭터/스타일만 간단히 확인
4. 컷별 이미지 자동 생성
5. 컷별 3초 영상 생성
6. 서비스가 자동 병합하여 18초 mp4 생성
7. 사용자가 다운로드/공유

### 시나리오 B. 경험자

1. 사용자가 기본 아이디어와 레퍼런스 이미지를 업로드
2. 생성된 플롯 중 일부 컷만 수정
3. 동일 캐릭터/동일 화풍으로 이미지 재생성
4. 시작 프레임/끝 프레임 강조하여 컷별 영상 생성
5. 최종 영상을 완성 후 공유

## 5. MVP 범위

자료 기준 필수 과제는 다음 3단계입니다.

* **AI 플롯 기획**: 아이디어 기반 컷 시나리오 생성
* **AI 이미지 생성**: 컷 간 캐릭터/스타일 일관성을 관리하는 이미지 생성
* **AI 영상 생성**: 이미지+텍스트 기반 3~5초 컷 영상 생성 및 연결
  추가 가산 과제는 **영상 병합/편집** 또는 **창작자 보상 체계** 중 택1인데, MVP 관점에서는 코어 플로우와 연결되는 **영상 병합/편집**을 채택하는 것이 가장 적합합니다. 

### 5-1. In Scope

1. 아이디어 입력
2. 컷 시나리오 자동 생성
3. 컷별 설정(등장인물/구도/배경/조명/스토리) 생성
4. 캐릭터 바이블/스타일 가이드 기반 이미지 생성, 선택적으로 첫 컷 레퍼런스 이미지 반영
5. 컷별 이미지 → 3~5초 영상 생성
6. 자동 병합하여 15~30초 결과물 생성
7. 결과물 미리보기 및 다운로드

### 5-2. Out of Scope

1. 커뮤니티 피드
2. 실시간 랭킹
3. 후원/결제
4. 정교한 타임라인 편집기
5. 고급 음향 편집
6. 다중 프로젝트 협업

## 7. 기능 요구사항

## 7-1. 기능 A: AI 플롯 기획

### 목적

자유 입력 텍스트를 분석해 5~10개 컷으로 구성된 시나리오를 생성한다. 자료상 필수 기능이다. 

### 입력

* 아이디어 문장 또는 키워드
* 선택: 장르/톤앤매너
* 선택: 목표 영상 길이(15초/30초)
* 선택: IP 이름 또는 태그

### 출력

컷 리스트(5~10개)
각 컷마다:

* 컷 번호
* 장면 설명
* 등장인물
* 감정/행동
* 구도
* 배경
* 조명
* 분위기
* 핵심 스토리 포인트

### 상세 요구사항

* 한 줄 입력만으로도 플롯 생성 가능해야 한다
* 컷 간 흐름이 자연스러워야 한다
* 숏폼 길이에 맞게 컷 수를 자동 조정한다
* 사용자가 컷 삭제/추가/순서 변경/텍스트 수정 가능해야 한다

### 수용 기준

* 사용자가 30초 이내에 첫 플롯을 받아볼 수 있다
* 최소 5개, 최대 10개 컷 생성 가능
* 각 컷에 필수 속성 5개 이상 자동 채워짐

---

## 7-2. 기능 B: AI 이미지 생성

### 목적

플롯의 각 컷을 시각 자료로 바꾸되, 캐릭터 외형과 화풍을 일관되게 유지한다. 자료상 필수 요구다. 

### 입력

* 컷별 장면 정보
* 공통 스타일 프리셋(애니/실사/웹툰풍 등)
* 선택: 캐릭터 레퍼런스 이미지
* 선택: 베이스 이미지
* 선택: 종횡비(9:16 기본)

### 출력

* 컷별 대표 이미지 1~4장
* 선택된 최종 이미지 1장

### 상세 요구사항

* 사용자는 복잡한 프롬프트 대신 옵션 선택만으로 생성 가능
* 동일 캐릭터의 헤어/의상/색상 특징 유지
* 동일 프로젝트 내 기본 화풍 유지
* 컷 단위 재생성 가능
* “이전 컷 스타일 유지” 옵션 제공

### 수용 기준

* 전 컷 이미지 생성이 가능해야 한다
* 사용자가 특정 컷만 다시 생성할 수 있다
* 최소한 “캐릭터 유지/스타일 유지” 체감이 있어야 한다

---

## 7-3. 기능 C: AI 영상 생성

### 목적

생성된 이미지와 컷 설명을 기반으로 3~5초짜리 컷 영상을 만들고, 이를 연결해 최종 숏폼을 제작한다. 자료상 필수 기능이다. 

### 입력

* 컷별 대표 이미지
* 컷 설명 텍스트
* 카메라 움직임 옵션(줌인/팬/정지)
* 길이 옵션(3초/5초)

### 출력

* 컷별 짧은 영상
* 최종 병합 영상(mp4/gif)

### 상세 요구사항

* 각 컷은 시작 프레임과 종료 프레임이 자연스럽게 느껴져야 한다
* 컷 간 연결이 지나치게 튀지 않아야 한다
* 사용자는 컷별 영상만 재생성 가능해야 한다

### 수용 기준

* 컷당 3~5초 영상 생성 가능
* 최소 3개 이상의 컷을 연결해 하나의 영상으로 저장 가능
* mp4 다운로드 가능

---

## 7-4. 기능 D: 영상 병합 및 편집 (가산 과제 선택)

### 선택 이유

자료에서 가산 과제는 두 가지지만, MVP 데모 완성도를 높이려면 **영상 병합 및 편집**이 가장 직접적입니다. 창작자 보상 체계는 서비스 확장 단계에서 붙이는 것이 더 적합합니다. 

### 제공 기능

* 컷 자동 순서 병합
* mp4 정규화 및 다운로드
* 썸네일 자동 추출
* 업로드용 제목/설명/해시태그 초안 생성
* YouTube 업로드 및 TikTok mock 업로드

### 수용 기준

* 최종 결과물을 15~30초 내로 출력
* 기본 편집 옵션 2~3개 제공
* 사용자가 수정 후 다시 내보내기 가능

## 11. API/백엔드 요구사항

### 11-1. 플롯 생성 API

**POST /projects/{id}/plot**

* input: idea_text, tone, target_duration
* output: cuts[]

### 11-2. 이미지 생성 API

**POST /cuts/{id}/images**

* input: cut metadata, style preset, reference image
* output: image candidates[]

### 11-3. 영상 생성 API

**POST /cuts/{id}/video**

* input: selected image, cut text, motion option
* output: video asset

### 11-4. 최종 렌더 API

**POST /projects/{id}/render**

* input: ordered cut videos, bgm option, subtitle option
* output: final mp4 url

## 12. 비기능 요구사항

### 성능

* 첫 플롯 생성: 30초 이내 목표
* 이미지 생성: 컷당 20~40초 내 체감 목표
* 영상 생성: 컷당 1~3분 이내 목표

* 컷 간 캐릭터 식별 가능해야 함

### 안정성

* 생성 실패 시 컷 단위 재시도
* 렌더 실패 시 마지막 성공 지점부터 복구

### 정책/안전

* 업로드 이미지 저작권 고지
* IP 기반 2차 창작물임을 안내하는 디스클레이머 표시
* 금지 소재 필터링

* 컷별 재생성 횟수

### 해커톤 데모 KPI

* 1회 데모에서 15~30초 결과물 완성
* 1명의 초보 사용자도 흐름을 이해 가능
* “툴을 여러 개 안 써도 된다”는 인식 확보

## 15. 리스크와 대응

### 리스크 1. 컷 간 일관성 부족

**대응:** 캐릭터 시트/스타일 가이드, 컷별 프롬프트 고정, 선택적 첫 컷 레퍼런스 이미지, 인접 컷 문맥 활용

### 리스크 2. 생성 시간 과다

**대응:** 컷 수 5~6개 기본값, 영상 길이 3초 기본값

### 리스크 3. 결과 품질 편차

**대응:** 프리셋 중심 UI, 사용자가 직접 프롬프트를 많이 쓰지 않도록 제한

### 리스크 4. 해커톤 기간 내 완성도 부족

**대응:** 편집 기능은 최소화, 커뮤니티 기능 제외, 코어 플로우에 집중

## 16. 권장 MVP 스펙(해커톤 현실안)

가장 현실적인 구현안은 아래입니다.

* **입력:** 한 줄 아이디어 + 스타일 선택
* **플롯:** 6컷 자동 생성
* **이미지:** 컷당 1장 생성, 필요 시 재생성
* **영상:** 컷당 3초 생성
* **병합:** 자동 페이드 연결
* **출력:** 18초 내외 mp4

이렇게 하면 자료의 필수 조건을 충족하면서도 데모 완성도가 높습니다.
